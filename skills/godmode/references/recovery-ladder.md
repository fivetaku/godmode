# 리커버리 사다리 — goaljaby 전달용 원문

아래 블록을 goaljaby 호출 프롬프트에 포함해 RECOVERY.md(및 goal-command의 종료 조건)에 반영을 요청한다.
goaljaby의 4,000자 컴팩트 예산을 존중해 필요 시 goaljaby가 축약하되, 표의 트리거/대응 쌍과 종료
조건은 보존되어야 한다.

---

## 골 종료 조건 (DoD)

이 골은 다음 두 조건을 **모두** 충족하기 전에는 완료로 선언하지 않는다:
1. VALIDATION의 모든 항목 통과 (증거 필수 — 실행 결과 없이 통과 처리 금지)
2. blocking 가정 0 (LOOP.md `blocking_assumptions_remaining` 참조)

DoD 미달 상태에서 작업이 끝나려 하면 아래 사다리 단계로 재진입한다. 정당한 멈춤은 세 가지뿐:
DoD 충족 / gate_pending(사용자 게이트 대기) / 사용자 명시 중단.

## 갭 리커버리 사다리

| 단계 | 트리거 | 대응 |
|---|---|---|
| 1 재시도 | 단발 실패 | 같은 접근으로 수정 후 재검증 |
| 2 접근 변경 | 같은 항목 2회차 연속 실패 | 접근 자체를 바꾼다 — 설정된 딥 리뷰어(LOOP.md `reviewer:`) 투입 또는 반박 관점 셀프리뷰 → 다른 방법으로 재시도 |
| 3 재계획 | 3회차 연속 실패 또는 구조적 갭(PRD 가정이 틀렸음이 드러남, 새 blocking 가정 발견) | LOOP.md에 `gate_pending`+방향 질문 내용을 기록하고 **골을 멈춘다** (골 도중 질문 금지 — 질문은 사용자가 재진입할 때 제시). 답을 받으면 PLAN/PRD 갱신 후 재진입 |

## 멈춤 원인 매핑 (전형적 조기 종료를 메꾼다)

| 멈춤 원인 | 처리 |
|---|---|
| 같은 에러 반복 → 포기하고 싶음 | 포기 금지 — 사다리 2단으로 강제 전환 |
| 도구·권한·환경 실패 | LOOP.md에 기록하고 우회 경로 시도, 우회 불가면 gate_pending |
| 모호해서 물어보고 싶음 | 자율 주행 중 질문 금지 — 합리적 기본값 채택 + PRD 가정 원장에 기록 후 계속 |
| "완료했다"는 느낌 | 느낌은 증거가 아니다 — VALIDATION 재실행 결과로만 done 판정 |

## 회차 기록

매 회차 LOOP.md에 블록 1개 append: 검증 결과 / 사다리 단계 / 사용 리뷰어 / 적용 태스크 / 판정.
세션이 중단되어도 LOOP.md가 재진입 지점이다.
